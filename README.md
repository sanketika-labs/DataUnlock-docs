# Data Unlock

**A trustworthy, federated architecture for turning fragmented data into AI-ready intelligence — without centralising it.**

Data Unlock is a Digital Public Infrastructure (DPI) framework designed to help governments, public institutions, and their technology partners convert siloed, human-readable datasets into interoperable, machine-interpretable, and AI-ready data products. It does this while preserving data ownership, enforcing governance, and enabling purpose-bound access.

## Core Idea

Most governments already *have* the data. What they lack is a mechanism to make that data trusted, linked, machine-readable, and accessible — in a way that respects institutional boundaries. Data Unlock provides that mechanism through a four-stage value chain:

1. **Prepare** — Standardise, certify, and catalogue raw data into trusted data products.
2. **Connect** — Resolve entities, align semantics, and enable federated queries across sources.
3. **Enable** — Transform data into AI-ready representations: embeddings, knowledge graphs, vector stores.
4. **Apply** — Deliver real-world outcomes through dashboards, conversational AI, and workflow automation.

Each stage is backed by open standards, open protocols, and (where available) open-source reference implementations.

## Key Concepts

- **Data Boarding Pass** — A declarative manifest that describes a specific Data Unlock deployment: who uses it, what data flows through it, for what purpose, and through what interfaces. [Learn more →](architecture/data-boarding-pass.md)

- **Data Passport** — The machine-readable metadata envelope that travels with a data asset as it moves through the ecosystem — combining provenance credentials, access policies, and structural metadata. [Learn more →](architecture/data-passport.md)
